# 🛡️ セキュリティ投稿ガイドライン

社員全体の意識向上とリスク低減に直結するため、以下のガイドラインに沿って投稿を行ってください。  
投稿の質を担保するため、各項目には記載ルールと承認基準を設けています。

---

## 📝 投稿時に必須の入力項目と記載ルール

| 項目 | 内容 | 制限・補足 |
|------|------|-------------|
| **題名（タイトル）** | 内容が一目で分かる具体的なタイトルを記載（例：生成AI利用時の注意点） | 必須 |
| **企業名** | 社外秘の場合は「某社」「A社」などに置き換える | ※投稿時には公開されません |
| **概要** | 投稿の目的や背景を簡潔に記載 | 100〜150文字程度 |
| **内容** | 事例・対策・考察などを具体的に記載 | 300〜600文字程度 |
| **メリット・デメリット** | セキュリティ対策のメリット・デメリットを記載 |  |
| **カテゴリタグ** | 「他社のセキュリティ対策事例」「最先端技術に関する情報」から選択 | 必須選択 |
| **技術難易度** | 初級／中級／上級から選択（読者の理解レベルに応じて） | 必須選択 |
| **参考URL** | 関連する社内Wikiや外部記事のリンクを添付 | 信頼性のある情報源のみ |

✅ 上記項目をすべて満たした投稿は、原則として承認対象となります。

---

## 🎯 ABC評価の目標について

投稿の内容に応じて、ABC評価の目標を立てることが可能です：

| 難易度 | 内容 | 内容の深さ |
|--------|------|-------------|
| **非常に高い** | 論文として提出できるレベルの内容で投稿する | 調査・分析＋根拠ある提言 |
| **高い** | レポートレベルの内容で投稿する | 複数事例＋具体的な施策案 |
| **普通** | 投稿ルールを満たす内容で投稿する | 簡潔な事例＋基本的な対策 |

📝 **補足：論文レベルの投稿について**  
内容が十分に体系的かつ専門性が高いと判断された場合、社外のセキュリティ関連学会（例：情報処理学会、JNSAなど）への論文提出を依頼することがあります。  
その際は、投稿者と相談のうえ、必要な編集・校正・共同執筆支援を行います。

---

## 🔐 セキュリティ関連投稿の注意点

- **具体性と再現性**  
  単なる注意喚起ではなく、具体的な事例や対処法を記載してください。  
  例：`USBメモリの紛失事例と社内対応フロー`

- **機密情報の取り扱い**  
  顧客名、個人情報、契約内容など、機密性の高い情報は記載しないでください。  
  必要に応じて匿名化・抽象化を行ってください。

- **誤解を招かない表現**  
  専門用語や略語は可能な限り解説を添え、誤解を招かないよう配慮してください。  
  例：`MFA（多要素認証）とは…`

- **最新情報の反映**  
  セキュリティは日々変化する領域です。投稿前に情報の鮮度を確認し、古い情報には  
  `※投稿時点の情報です` と明記してください。

---

## ✅ 投稿承認の流れ（必須プロセス）

すべての投稿は、以下の承認プロセスを経て初めて掲示板に掲載されます。  
**承認を受けていない投稿は掲載できません。**

1. 投稿者がガイドラインに沿って記載  
2. 管理者が内容・記載ルール・機密性を確認  
3. 承認 → 掲示板掲載  
4. 論文レベルの場合 → 学会提出の可能性あり（別途連絡）
